What is Syslog in Linux?

Syslog is a standardized logging protocol and service used in Linux systems to collect, process, and store log messages 
generated by the operating system, applications, and services. It provides a centralized way to manage logs, making 
troubleshooting and monitoring more efficient.

Why Use Syslog?

Centralized Logging: Aggregates logs from multiple sources into a single location.

Severity Levels: Categorizes logs by urgency (e.g., emergency, error, warning).

Facilities: Organizes logs by source (e.g., kernel, mail, auth).

Flexibility: Logs can be stored locally, forwarded to remote servers, or processed by tools like Elasticsearch.

Automation: Enables log rotation and archival to prevent disk space issues.

Key Components
--------------

Syslog Daemon:

	The service handling log collection (e.g., rsyslog or syslog-ng).

Configuration Files:

	/etc/rsyslog.conf: Main configuration file.

	/etc/rsyslog.d/*.conf: Additional custom rules.

Log Files:

	Stored in /var/log/ (e.g., syslog, auth.log, kern.log).

Scenario Without Syslog:
------------------------
Problem: If 5 apps write logs to separate files (e.g., /var/log/app1.log, /var/log/app2.log, etc.), you must check all 5 files individually for errors.

Pain Point: Manual effort to monitor multiple files.

Scenario With Syslog:
---------------------
Solution: Syslog can centralize logs from all 5 apps into a single file (or a few categorized files), depending on your configuration.

Advantage: You can monitor one consolidated log (e.g., /var/log/syslog) for errors from all apps, but only if:

The apps are configured to send logs to syslog (not to their own files).

Syslog rules are set to aggregate logs into a single file.
